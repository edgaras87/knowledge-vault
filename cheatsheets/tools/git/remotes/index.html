üìù Git Local ‚Üî Remote Cheat Sheet¶

üîë Core ideas¶

  • clone ‚Üí make a new local folder from remote
  • init ‚Üí make current folder a repo
  • remote = nickname + URL (origin)
  • upstream = default remote branch your local branch tracks

üöÄ Common Scenarios¶

1. Start from Remote ‚Üí Local¶

‚úÖ Now just git pull and git push

2. Start from Local ‚Üí Remote¶

The -u flag sets the upstream for your branch. It tells Git: ‚Äúmain should track origin/main from now on.‚Äù

3. Already Have Both (Sync Them)¶

üåç Understanding ‚ÄúUpstream‚Äù¶

The upstream branch is what your local branch ‚Äútracks.‚Äù When you set it, git pull and git push automatically know which remote branch to sync with ‚Äî no need to retype origin main every time.

git push -u origin main   # set upstream during first push
git branch --set-upstream-to=origin/main main  # set manually

After setting upstream:

git push     # pushes to origin/main automatically

If you ever clone a repo, upstreams are usually set for you. You‚Äôll only need to set them manually when creating new branches or pushing from a freshly inited local repo.

Think of it this way: origin is the remote name (like a server nickname). origin/main is the remote branch. The upstream is your local branch‚Äôs connection to it.

üìÇ Folder Rules¶

  • git clone <url> ‚Üí makes a new folder
  • git clone <url> myfolder ‚Üí makes myfolder
  • git clone <url> . ‚Üí current empty folder
  • Always run Git inside the repo folder

ü§î Which First?¶

  • Solo / quick test: local ‚Üí push
  • Team / templates / CI: remote ‚Üí clone

‚ö° Handy Commands¶
