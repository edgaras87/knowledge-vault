üß† Philosophy¶

  1. Organize by capability, not dependency.
    Jump straight to ‚Äútransactions‚Äù or ‚Äúsecurity‚Äù without recalling which starter or module they live in.

  2. Keep pages thin. Each .md covers one concept cluster (e.g., mappings, transactions).
    Avoid giant scrolls; prefer clear entry points.

  3. Cross-link ruthlessly. Example:

  4. @Transactional ‚Üí links to data/queries.md

  5. @PreAuthorize ‚Üí links to security/method-security.md and core-di/conditions.md

  6. Reactive-ready. WebFlux lives in its own corner but reuses MVC annotations; highlight differences, don‚Äôt duplicate content.

  7. Boot awareness without dependency. Boot annotations live under /boot, but pages cross-reference corresponding Core features.

üóÇ Canonical Annotation Sets¶

@Component, @Service, @Repository, @Controller, @Configuration, @Bean, @Import, @Scope, @Lazy, @Autowired, @Qualifier, @Primary, @Value, @Profile, @Conditional, @PostConstruct, @PreDestroy, @EventListener

MVC / REST¶

@Controller, @RestController, @ControllerAdvice, @RequestMapping, @GetMapping, @PostMapping, @PutMapping, @DeleteMapping, @PathVariable, @RequestParam, @RequestBody, @ResponseBody, @ResponseStatus, @ExceptionHandler, @CrossOrigin

Validation¶

Data¶

Security¶

Async / Scheduling¶

@Async, @EnableAsync, @Scheduled, @EnableScheduling

@EnableCaching, @Cacheable, @CachePut, @CacheEvict, @Caching

Messaging¶

Actuator¶

Boot¶

Spring Annotations ‚Äî Master Map¶

Jump by capability:

  • Core & DI ‚Üí Stereotypes ¬∑ Beans ¬∑ Profiles
  • MVC / REST ‚Üí Controllers ¬∑ Mappings ¬∑ Errors
  • WebFlux ‚Üí Enable & Config
  • Validation ‚Üí @Valid & @Validated
  • Data ‚Üí Repositories ¬∑ Queries
  • Transactions ‚Üí @Transactional
  • Security ‚Üí Method Security
  • Async / Scheduling ‚Üí @Async ¬∑ @Scheduled
  • Caching ‚Üí Cache annotations
  • Messaging ‚Üí Kafka ¬∑ RabbitMQ
  • Actuator ‚Üí Custom endpoints
  • Boot ‚Üí Auto-Config ¬∑ ConfigurationProperties

Legend: Each page starts with ‚ÄúEssence / When to use / Gotchas‚Äù and ends with ‚ÄúTroubleshooting / See also‚Äù.

üß© Maintenance Discipline¶

  • One page = one concept cluster. Avoid ‚Äúeverything about @Controller in 500 lines‚Äù pages.

  • Short code, clear output. Examples must compile and demonstrate only the annotation‚Äôs core purpose.

  • Cross-references are mandatory. Use links liberally to show relationships and dependencies.

  • Track freshness. Use last_update: and framework version tags for periodic review when Spring 7+ arrives.
